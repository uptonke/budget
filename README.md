# 年度娛樂預算

純前端年度娛樂預算工具，可直接上傳到 GitHub repo 或用 GitHub Pages 部署。

## 檔案

```text
index.html
style.css
script.js
README.md
```

## 功能

- 自訂年度基礎預算、年度調整、年度備註
- 自訂年度計畫：分類、項目、次數、單價、備註
- 自訂分類預算上限
- 分類使用率進度條
- 超支警示：年度爆表、分類超支、計畫超限、接近上限
- 計畫 / 實際差異表
- 實際流水帳：支出、退款 / 收入
- 用實際花費計算年度結餘 / 赤字
- 赤字會自動結轉到下一年
- 年度結轉預覽
- 複製上一年度計畫與分類上限，不複製實際流水帳
- 匯出 JSON，檔名自動帶年度與日期時間
- 匯入 JSON
- 刪除年度、編輯 / 刪除計畫項目、編輯 / 刪除實際流水帳、編輯 / 刪除分類上限

## 使用方式

1. 打開 `index.html`。
2. 在「年度設定」輸入年度基礎預算。
3. 按「加入基準版」或自行新增年度計畫。
4. 在「分類預算上限與差異分析」設定各分類年度上限；也可以按「用年度計畫建立上限」。
5. 實際花錢後，在「實際流水帳」新增紀錄。
6. 定期按「匯出 JSON」備份。

|---|---|---:|---:|

| 爵士 | Bluenote / 爵士酒吧 | 12 | 550 |

## 結轉規則

年度可用總額：

```text
年度基礎預算 + 上年結轉 + 年度調整 + 實際收入 / 退款
```

年度期末：

```text
可用總額 - 實際支出
```

下一年上年結轉：

```text
上一年度期末結餘 / 赤字
```

結轉只看「實際流水帳」，不看年度計畫。

## localStorage 限制

資料存在瀏覽器 `localStorage`。這代表：

- 換裝置不會自動同步
- 清除瀏覽器資料可能會遺失
- GitHub Pages 無法直接把資料寫回 repo

建議定期匯出 JSON 備份。需要跨裝置同步時，再升級到 Supabase 或其他後端。
