# 娛樂預算 T 字帳 v2

一個純前端的年度娛樂預算紀錄工具。可直接部署到 GitHub Pages。

## 功能

- 自訂年度基礎預算
- 自訂年度調整金額
- 自訂年度計畫項目、次數、單價
- 記錄實際流水帳：支出、退款 / 收入
- 用「實際花費」自動計算年度期末結餘 / 赤字
- 赤字也會自動結轉到下一年
- 會計式 T 字帳視覺：Debit 用途 / Credit 來源
- 年度總覽
- 匯出 / 匯入 JSON 備份
- 一鍵加入基準版模板

## 預設基準版模板

| 項目 | 次數 | 單價 | 年預算 |
|---|---:|---:|---:|
| 23 全英文喜劇 | 12 | 600 | 7,200 |
| Bluenote / 爵士酒吧 | 12 | 550 | 6,600 |
| fine dining / omakase | 4 | 4,000 | 16,000 |
| 過年遠程出國 | 1 | 55,000 | 55,000 |
| 東南亞連假 | 1 | 12,500 | 12,500 |

基準版合計：97,300。

## 結轉規則

系統使用「實際流水帳」計算結轉，而不是計畫表。

```text
可用總額 = 年度基礎預算 + 上年結轉 + 年度調整 + 實際收入 / 退款
期末結餘 = 可用總額 - 實際支出
下一年上年結轉 = 前一年期末結餘
```

若期末為赤字，下一年會自動以負數結轉。

## 部署到 GitHub Pages

1. 解壓縮 zip 檔。
2. 把 `index.html`、`style.css`、`script.js`、`README.md` 上傳到你的 repo 根目錄。
3. 到 GitHub repo：`Settings` → `Pages`。
4. Source 選擇你的 branch，例如 `main`。
5. Folder 選 `/root`。
6. 儲存後等待 GitHub Pages 部署完成。

## 重要限制

這是純前端靜態網站，資料存在瀏覽器 `localStorage`。

代表：

- 同一台裝置、同一個瀏覽器會保留資料
- 換手機、換瀏覽器、清除瀏覽器資料，資料不會自動同步
- GitHub Pages 不會自動把資料寫回 repo

建議定期按「匯出 JSON」備份資料。

## 何時需要 Supabase

目前 v2 不接 Supabase。等你需要以下功能再接：

- 手機 / 電腦跨裝置同步
- 登入帳號
- 遠端永久備份
- 多人共用
- 想把資料當成真正資料庫查詢
